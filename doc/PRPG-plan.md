## 1. Architectural Context & Design Rationale

**The ORCA Objective:** The platform’s reinforcement learning (RL) agent must learn the optimal asset location strategy (e.g., placing tax-exempt municipal bonds in taxable accounts, and corporate bonds in tax-deferred accounts) to maximize after-tax risk-adjusted returns.

**Tax Handling vs. Path Generation:** It is explicitly noted that the ORCA environment natively handles all specific tax treatments (e.g., separating capital gains from interest returns). The PRPG process outlined in this document is strictly responsible for handling the total return path generation.

**Why HMM-Conditioned Multivariate Bootstrap:**
If you feed an RL agent synthetic data generated by a Vector Error Correction Model (VECM), the agent will quickly detect the deterministic mean-reversion formula. Instead of optimizing for tax efficiency, the agent will learn to "time" the mean reversion, transforming into an alpha-seeking strategy. This pollutes your study with speculative pre-tax alpha.

The **HMM-Conditioned Multivariate Bootstrap** solves this by completely removing deterministic price drift.

1. The **Hidden Markov Model (HMM)** accurately simulates the duration and transitions of macroeconomic regimes.


2. The **Multivariate Bootstrap** draws historical return blocks randomly within those regimes. Because the sequence of blocks is randomized, the RL agent cannot predict the next price movement. It is forced to abandon market timing and focus exclusively on risk management, asset location, and tax-loss harvesting to generate value.



To prevent look-ahead bias and inadvertent foresight, the RL agent does **not** observe the macro variables or the HMM state. The agent only sees asset prices, asset returns, and its own portfolio state.

## 2. Data Acquisition & Processing Strategy

To ensure reproducibility, we utilize `yfinance` for asset returns and the Federal Reserve Economic Data (FRED) API for macroeconomic indicators.

### A. Asset Class Data (`yfinance`)

We will pull the adjusted close prices (which account for dividend reinvestment) to calculate log returns.

* **Global Equity:** `ACWI` (iShares MSCI ACWI ETF).


* **Taxable Fixed Income:** `AGG` (iShares Core US Aggregate Bond ETF).


* **Tax-Exempt Fixed Income:** `MUB` (iShares National Muni Bond ETF).



**Study Horizon & Data Integrity:** We will strictly shorten the study horizon to the post-2007/2008 era to exclusively use actual ETF data. Extending the history back to the 1990s using active mutual fund proxies (such as `VWITX` or `VHGEX`) is strictly avoided. Utilizing mutual funds introduces differing duration exposures, credit qualities, management alpha, survivorship bias, and style drift. Furthermore, concatenating proxy series at the ETF inception date produces statistical jumps in volatility and yield spreads that the HMM may erroneously treat as a regime change.

**Frequency Resampling:** Fetch daily data and compute log returns. Use Pandas to build consistent timeframes.

### B. Macroeconomic Data (FRED API via `pandas_datareader`)

To prevent the HMM from simply memorizing equity market crashes, **do not feed asset prices into the HMM**. Feed it pure macroeconomic data that dictates the underlying economic cycle.

1. **Industrial Production (`INDPRO`):** Monthly. Measures real output; the cleanest proxy for economic growth/recession. (Transform to Month-over-Month % change).


2. **Inflation (`CPIAUCSL`):** Monthly. Consumer Price Index. Crucial for bond yield modeling. (Transform to Year-over-Year % change).


3. **Yield Curve Spread (`T10Y3M`):** Daily. 10-Year minus 3-Month Treasury spread. The premier indicator for impending economic contractions.


4. **Credit Spread (`BAA10Y`):** Daily. Moody's Baa Corporate Bond Yield minus 10-Year Treasury. Captures systemic credit risk and liquidity stress.

## 3. HMM Implementation (Regime Identification)

The HMM will classify historical data into distinct market states, which serves as the "conditioning" for the bootstrap.

1. **Feature Engineering:** Standardize the four macro variables above into z-scores so they operate on the same scale. Align them all to a monthly frequency.


2. **Model Configuration:** Use `hmmlearn.hmm.GaussianHMM` in Python.


* Set `n_components=3` to represent three intuitive states: **Expansion** (growth, steep yield curve), **Contraction** (shrinking production, tight credit), and **Stagnation** (high inflation, flat growth).


* Set `covariance_type="full"` so the model captures the covariance between inflation and yield spreads.




3. **Historical Decoding & Smoothing:** Run the Viterbi algorithm to decode the historical timeline. To prevent the generation of instantaneous regime switches—where one month's return vector jumps abruptly to a completely different distribution—implement smoothing on the state sequence or utilize a regime-switching model with autoregressive dynamics within regimes. This ensures transitional dynamics mimic the gradual way financial markets anticipate macro shifts.

## 4. Multivariate Stationary Bootstrap Design

With the historical timeline tagged by regime, you can generate synthetic paths.

### Step 1: Simulate the Macro Sequence

Use the transition matrix $A$ learned by the HMM to simulate a new sequence of states. This ensures realistic regime durations.

### Step 2: The Stationary Bootstrap Resampling Algorithm

To eliminate ambiguity and prevent edge effects, the resampling process is formally specified as follows:

* For each state-segment of length $D$, randomly draw a starting index from the historical data that matches that specific state.


* Independently draw block lengths $L$ from a geometric distribution, $P(L=k) = (1-p)^{k-1}p$, until the $D$ months are completely covered.


* Stitch these independently generated blocks consecutively to fill the state's duration.



### Step 3: Multivariate Enforcement

When you pull a block of returns for a specific timeframe, you **must pull the returns for ACWI, AGG, and MUB simultaneously** as a single vector. This perfectly preserves the historical covariance matrix, ensuring your RL agent experiences exactly how Muni bonds reacted relative to equities during specific credit events.

## 5. Calibration & Parameter Tuning

* **Tuning HMM States ($N$):** Fit models for $N \in [2, 3, 4, 5]$ and calculate the Bayesian Information Criterion (BIC). Choose the $N$ that minimizes BIC. Overfitting $N$ will result in states with too few historical observations, breaking the bootstrap.


* **Tuning Block Size ($p$):** The parameter $p$ dictates the expected block length ($\mathbb{E}[L] = 1/p$).


* Use the **Politis-White automatic block-length selection algorithm**. For monthly data, you typically calibrate $p$ such that the expected block length is roughly 3 to 6 months.

## 6. Measures of Success & Quality Gates

Before allowing the RL agent to train on the simulated data, the environment must pass three strict quality gates.

### Gate 1: The Pre-Tax Neutrality Test (Alpha Leakage)

* **The Test:** Deploy several common alpha-seeking dummy algorithms (e.g., momentum, short-term reversal, and carry strategies) across a large number of simulated paths. Ensure these bots only see price history, not the hidden states.


* **Success Metric:** The dummy algorithms' pre-tax Sharpe ratios must not be statistically significantly different from a static buy-and-hold benchmark. If the bots achieve a high Sharpe ratio, the bootstrap boundaries are leaking predictable patterns.



### Gate 2: Covariance Matrix & Autocorrelation Preservation

* **The Test:** Calculate the historical covariance matrix $\Sigma_{hist}$ and the simulated covariance matrix $\Sigma_{sim}$ averaged across the generated paths. Additionally, execute an autocorrelation check to verify the internal autocorrelation structure matches historical data.


* **Success Metric:** Calculate the Frobenius norm of the difference ($\vert{}\vert{}\Sigma_{hist} - \Sigma_{sim}\vert{}\vert{}_F < \epsilon$). The threshold $\epsilon$ must be strictly calibrated from the sampling variability of the historical covariance estimate itself to avoid rejecting statistically indistinguishable paths.



### Gate 3: Asset Location Spread Accuracy (Wasserstein Distance)

* **The Test:** The core of ORCA's logic hinges on the yield/return spread between taxable and tax-exempt assets.


* **Success Metric:** Measure the Wasserstein metric (Earth Mover's Distance) on the **full joint distribution** of returns—not just the marginal spread—to guarantee that tail dependencies are preserved. The simulated distribution must perfectly overlay the historical distribution.
